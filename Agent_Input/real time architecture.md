**Tech Stack**

| Component     | Recommended Tool                                 | Justification                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|---------------|--------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Ingestion     | Snowpipe Streaming                               | Supports high-velocity, low-latency CDC/event ingestion from upstream sources once Bronze is redesigned. Micro-batch and streaming loads from external sources are natively supported.                                                                                                                                                                                                                                                                                                                                                         |
| Processing    | Snowflake Dynamic Tables; Streams & Tasks        | Dynamic Tables support continuous/micro-batch transformation, deduplication, and upsert patterns required for Silver and Gold. Streams & Tasks orchestrate incremental stateful logic, windowed aggregations, and change capture.                                                                                                                                                                                                                                                                                                               |
| Storage       | Snowflake Native Tables                          | All curated and intermediate data (Silver, Gold, Audit layers) stored in native Snowflake tables, supporting both streaming and micro-batch writes.                                                                                                                                                                                                                                                                                                                                                      |
| Orchestration | Snowflake Tasks (or external orchestrator if needed) | Tasks coordinate micro-batch windows, deduplication, and upsert logic, and enable dependency-aware sequencing. Parallelism and event-driven triggers are supported for decomposed procedures. External orchestration may be considered for cross-platform or complex dependencies, but not required for the described patterns.                                                                                                                                    |
| Monitoring    | Snowflake Alerts, Data Metric Functions          | Platform-native alerting and data metric functions monitor streaming pipeline health, latency, and data quality, supporting operational SLAs for real-time and micro-batch workloads.                                                                                                                                                                                                                                                                                                                   |
| Data Quality  | Snowflake Data Metric Functions; Audit Tables    | Data quality rules (row-level validation, deduplication, completeness) implemented as part of Dynamic Table/Streams logic and monitored via Data Metric Functions. Audit tables (populated by usp_Log_Data_Quality_Error, usp_Log_Pipeline_Audit) support traceability and compliance.                                                                                                                                    |
